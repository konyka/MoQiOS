M14: SMP Status (Current Work)

What Works

  • ACPI MADT parsing — detects 2 CPUs, LAPIC IDs
  • 3-stage AP trampoline: real → protected → long mode
  • Identity mapping covering all 512MB RAM (256 PD entries)
  • Per-CPU GDT/TSS initialization
  • Per-CPU data: percpu_array[N] with cpu_id, apic_id, current_tid
  • GS base MSR configured per CPU
  • AP enters idle loop (sti; hlt)
  • 26/26 tests pass with -smp 2

Known Limitations

LAPIC MMIO on AP: Accessing LAPIC MMIO (HHDM or identity-mapped) crashes the AP in QEMU TCG. Tried: 2MB huge page, 4KB page, identity mapping, MSR 0x1B enablement — all fail. AP runs without LAPIC timer interrupts (uses sti; hlt idle loop).
Kernel spinlocks integrated: Serial, PMM, task array, and scheduler now use IrqSpinlock. IPC and syscall handlers still modify task state directly (safe: only BSP runs syscalls). Lock order: sched_lock → task_lock → pmm.lock. CR3 reloads done outside lock.

Key Technical Decisions

  • Identity-mapped entry point: AP's TLB is empty; using physical address for apEntry avoids walking kernel mapping (PML4[511]) for instruction fetch
  • CPU ID via @ptrFromInt(0x7040): Zig callconv(.c) calling convention unreliable; reading from trampoline data area avoids register mismatch
  • Precompiled flat binary trampoline: as --32 + ld -melf_i386 + objcopy -O binary
  • Zig 0.16 register clobbers: Use .{ .rax = true } struct syntax, NOT "rax" strings
